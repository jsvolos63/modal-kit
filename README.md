# @jfs/modal-kit

Shared, dependency-free **accessible-dialog plumbing** for the JFS family of
buildless static PWAs: focus trap + focus save/restore, iOS-safe scroll-lock, a
central Escape stack, marker-guarded `inert`/`aria-hidden` siblings, bfcache
cleanup, and an opt-in history sentinel so the browser Back button (and iOS
edge-swipe) closes the topmost dialog.

Six repos hand-roll this, from best-in-class to buggy. Bears' `js/lib/modal.js`
has the robust *environment* layer (`position:fixed` scroll-lock with offset
restore, marker-guarded inert siblings, soft-keyboard blur, `pagehide` cleanup)
but no Tab trap; JFS-Sports' `modal-focus.js` and Art-Gallery's
`createModalSession` have the *lifecycle* layer (a real focus trap, a
reference-counted open stack, history-back close). This package is the promoted
superset of the two.

## Usage

```js
import { createModal } from './modal-kit/index.js';

const modal = createModal(document.getElementById('sheet'), {
  focusTarget: '#sheet-close',
  onClose: () => resetForm(),
});
openBtn.addEventListener('click', () => modal.open());
```

`createModal(el, options)` returns `{ open(), close(), isOpen() }`. One call per
dialog; the module keeps a shared stack so Escape and the Back button always act
on the topmost open dialog, and scroll-lock is reference-counted across the
stack. A falsy `el` returns an inert no-op controller.

### Options

| Option | Default | Effect |
|---|---|---|
| `focusTarget` | first focusable | element, selector-within-`el`, or default to the first focusable (falls back to focusing `el` itself with `tabindex=-1`) |
| `focusDelay` | `0` | ms to defer initial focus (set ~`30` to let iOS finish hiding the soft keyboard, as Bears does) |
| `escClose` | `true` | Escape closes this dialog when it's topmost |
| `trapFocus` | `true` | wrap Tab / Shift+Tab inside `el` (attached per-dialog, so nested dialogs each trap correctly) |
| `scrollLock` | `true` | lock body scroll while open |
| `scrollLockClass` | `'modal-open'` | the body class that supplies `position:fixed` |
| `inertSiblings` | `true` | mark sibling elements `inert` + aria-hide them, restoring only what it set |
| `closeOnBackdrop` | `true` | a pointer on `el` itself or any `[data-close]` element closes it |
| `shouldCloseOnPointer(e)` | — | replace the default backdrop / `[data-close]` predicate |
| `history` | `false` | push a history sentinel so Back / edge-swipe closes it |
| `hiddenAttr` | `true` | toggle `el.hidden` for visibility |
| `openClass` | — | also toggle this class on `el` (for CSS that keys visibility off a class, e.g. `is-open` / `visible`) |
| `ariaHidden` | `true` | toggle `el`'s `aria-hidden` with visibility |
| `onOpen({el})` / `onClose({el})` | — | lifecycle callbacks |

Also exported: `isAnyModalOpen()`, `getFocusable(container)` (the same
visible-focusable definition the trap uses), and `_resetModalsForTest()`.

### Required CSS

Scroll-lock pins the body with `position: fixed` and a negative `top` offset
(set via the CSSOM, so a strict CSP `style-src` without `'unsafe-inline'` still
allows it). Ship the class in your stylesheet:

```css
.modal-open { position: fixed; width: 100%; }
```

The offset is restored (and the page scrolled back) when the last dialog closes.

## How it's consumed

These are buildless sites — `node_modules` is not deployed. Each app commits a
generated copy of `index.js` (generated by the kit's own vendoring CLI) and
imports it directly. ES-module apps vendor the verbatim ESM copy:

```json
"vendor:sync":  "jfs-modal-kit-vendor --format esm --out modal-kit/index.js",
"vendor:check": "jfs-modal-kit-vendor --format esm --out modal-kit/index.js --check"
```

Classic-script apps use `--format global --name ModalKit` (exposes
`globalThis.ModalKit`) or `--format bare` where the file is concatenated into a
bundle. The exposed surface is derived from `index.js`'s own `export`
declarations, never a hand-maintained list; `npm run vendor:check` fails CI on
drift.

## Not in scope (v0.1.0)

Art-Gallery's **swipe-to-dismiss** gesture is intentionally left out of the base
— it's pointer-heavy and app-specific; an adopter that needs it keeps its local
gesture layer on top of this kit for now.

## Test


real-ish DOM via `jsdom` (a devDependency), plus the family's shared vendor-shim
smoke test.
